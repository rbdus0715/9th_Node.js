- 서버에서 내려주는 응답의 크기가 크면 네트워크 대역폭을 많이 사용하고, 전송 속도 역시 느려질 수 있습니다. 이를 위해 일반적으로는 `gzip` 등의 방식의 압축을 통해 응답을 압축하여 제공하기도 하는데요, `gzip`, `brotli` 등의 압축 방식들에 대해 정리하고 + Express 프로젝트에 응답을 압축하는 미들웨어를 추가해주세요. (단, 너무 작은 응답까지 불필요하게 압축하지 않도록 `threshold` 값을 `0.5kb(512b)`로 설정해주세요.)
    
    ## 1. Gzip
    
    - 정의: 데이터를 압축해서 전송하는 오래된 방식
    - 원리: 중복된 데이터를 찾아서 줄이고, 작은 형태로 변환해서 보냄
    - 장점:
        - 거의 모든 브라우저와 서버에서 지원
        - 구현이 쉽고 안정적
    - 단점:
        - 압축률이 Brotli보다 낮음 (즉, 압축 후 데이터 크기가 더 클 수 있음)
    - 사용 예시: HTML, CSS, JS, JSON 등 웹에서 자주 사용되는 텍스트 파일 압축
    
    ---
    
    ## 2. Brotli
    
    - 정의: 구글에서 만든 최신 압축 방식
    - 원리: 더 똑똑하게 데이터를 압축 → Gzip보다 작은 크기로 만들 수 있음
    - 장점:
        - Gzip보다 압축률이 높음 → 데이터가 더 작아져서 전송 속도 빨라짐
        - 특히 텍스트 기반 파일에 효율적
    - 단점:
        - 일부 오래된 브라우저에서는 지원하지 않음
        - 압축할 때 CPU 사용량이 약간 더 높음
    - 사용 예시: 최신 브라우저에서 제공하는 정적 파일 최적화 (HTML, CSS, JS, JSON)
